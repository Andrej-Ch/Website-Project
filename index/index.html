Question 2

IPv6 was introduced because the amount of IPv4 addresses is becoming exhausted. IPv4 uses 32-bit addresses which provide ~4.3 billion addresses, which is clearly not enough, as of December, 2014, IPv4 still carries 94% of the internet`s traffic. Something needed to be done so that there would be more IP addresses available, hence IPv6 was introduced. IPv6 uses 128-bit addresses which would be roughly enough to give an IP address to every particle in our universe. IPv6 does not just introduce a larger amount of IP addresses available, it also has some technical upgrades. IPv6 permits hierarchical address allocation methods that facilitate route aggregation across the Internet, and so limit the expansion of routing tables. The use of multicast addressing is expanded and simplified, and provides additional optimization for the delivery of services. Device mobility, security, and configuration aspects have been considered in the design of the protocol. The issue which is making the transition from IPv4 to IPv6 difficult is that IPv6 was not designed to be interoperable with IPv6, but there are many ways being explored how to ease this transition. One is that an IPv4 packet could carry an IPv6 packet inside of it, using IPv4 as a link layer for IPv6, this is called Tunneling. Another method is called Dual IP stack implementation, it provides that IPv4 and IPv6 can both run side by side using the same network infrastructure and this is the most desirable way.

Question 4

Idempotence:

Methods having the property of idempotence means that (aside from error or expiration issues) the side-effects of N > 0 identical requests is the same as for a single request. The methods GET, HEAD, PUT and DELETE share this property. Also, the methods OPTIONS and TRACE SHOULD NOT have side effects, and so are inherently idempotent.

Simplifying HTTP proxy caching:

The basic purpose of HTTP caching is to provide a mechanism for applications to scale better and perform faster. But HTTP caching is applicable only to idempotent requests, which makes a lot of sense; only idempotent and nullipotent requests yield the same result when run multiple times. In the HTTP world, this fact means that GET requests can be cached but POST requests cannot.

Visitor counters:

Since the result of the GET method is exactly the same after each time it is performed, the results can easily be cached. By implementing a visitor counter, the result will actually be different from other requests, so you shouldn't cache it.

Negative effect from proxies:

A proxy allows multiple visitors access the page using the same IP address, they are seen as the same visitor. The consequence of this is that the visitor counter is not increased.

Question 3

Internationalization is achieved when the design of content, application, specification etc., provides easy access and usage to users who come from different cultures and regions and speak different languages. Internationalization raises several issues to be addressed for web designers and developers. Making a web application comprehensible for people who speak different languages can be a challenging task. Although Unicode, which is the main computing industry standard behind W3C technologies, allows the storage of different characters and symbols in a slightly different order, it cannot ensure the canonical equivalence of the resulting sequences. When the key issue of canonical equivalence is not handled, it may lead to incorrect parsing, comparing and sorting of text. Additionally, different writing systems require different text alignment. For example, arabic is written and read from right to left. A combination of text alignment and other typographic conventions should be supported in the style sheets of a mark up language. A mark up language should also allow the input and representation of different formats of data, such as alternative calendars. The translatable content should be comprehensible. Different languages use different syntax. As a result, the web appications should not be restricted to an english based syntax. Cultural issues may occur next to the above mentioned linguistic problems. Symbolism is an example of cultural controversy. (e.g. a checkmark could indicate something incorrect in Japanese culture.)

Without Net Neutrality, your Internet Service Provider could arbitrarily block whatever websites it didn't want you to access (e.g. perhaps blocking you from accessing competitors' websites).   1. Apple insists that they want to give their users a choice, to purchase e-newspaper through iTunes, or directly from newspapers’ own websites. ENPA is worried that Apple will allow users to buy papers directly from publishers, and this would largely cut into newspapers profit.
  2. Allowing consumers to only have access to the newspaper of their choice via the iTunes store, is a major violation of net neutrality principals, as all data should be treated equally regardless of how user gains access to it.

Question 8

Differently from iTunes Spotify does not allow user to download music, but rather streams it online. It could be a great way for upcoming artists to get recognition and some financial gain, however they aren’t as supportive to artists as they claim to be.

Spotify pays musicians ~0.5 cent per play ($5,000 per million plays) and they have received a lot of criticism over the years, because the biggest piece of cake goes to major labels and recording companies via licensing fees (over US$1bn paid out by 2013), while artists are struggling.

Spotify should open their own independent record company where they would collaborate directly with artists, so cutting out the middle man – major record labels. This way Spotify could save large amounts of money on licensing, and this would allow the artists to get paid what they deserve.

Question 9

Some people are more susceptible to false news, therefore reducing hoax articles on Facebook can only have positive effects. It will reduce the amount of misinformation floating in social media and will punish people profiting from it.

Question 10

The purpose of data.gov is to increase public access to high value, machine readable datasets generated by the Executive Branch of the Federal Government." The site seeks to become "a repository for all the information the government collects". The site would publish to the public any data that is not private or restricted for national security reasons.

It’s a great example of “transparent government”, but like any other medium it can be manipulated for financial or political purposes and shouldn’t be trusted blindly.
